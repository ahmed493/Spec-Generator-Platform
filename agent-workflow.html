Pipeline Multi-Agents — Génération de Spécifications

Workflow automatisé de bout en bout piloté par des agents IA spécialisés

Upload du
template PDF · Markdown
01
Template Agent
Analyse et parsing du template
  • Lecture PDF ou Markdown, extraction complète du texte
  • Détection de tous les champs et sections du template via LLM
  • Classification : texte, liste, table, date, paragraphe
Output
100 % des champs + sections détectés
02
Extraction Agent
Extraction des données sources
  • Lecture intégrale du repo : SQL, Python, YAML, JSON, notebooks
  • Injection du contexte live (BigQuery, PostgreSQL, GCS, Power BI)
  • Correspondance champ ↔ valeur par LLM + indicateurs de confiance
Output
Dictionnaire {field_id → valeur} enrichi
03
Mapping Agent
Composition de la spécification
  • Remplissage du template avec les valeurs extraites
  • Préservation de la structure Markdown d'origine
  • Marquage ❓ pour les champs non identifiés
Output
Spécification complète (Markdown)
04
Validation Agent
Contrôle qualité et approbation
  • Vérification de tous les champs obligatoires
  • Détection des valeurs suspectes ou incomplètes
  • Rapport de validation + approbation utilisateur
Output
Spec validée, prête à l'export
